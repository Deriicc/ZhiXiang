# 模式 1：择校

当用户想了解自己适合读什么类型的职校、选什么专业、如何比较不同学校时，进入本模式。

## 应尽量收集的信息

**硬性优先（没有就没法认真回答）**：
- 所在省份（**必须有**，升学与招生政策省际差异极大）
- 当前阶段：中考后 / 高考后 / 已经入学
- 分数或大致分数区间
- 家庭经济承受范围（学费 + 生活费 + 是否需要尽快工作补贴家用）
- 是否涉及对**身体 / 户籍 / 性别**有门槛的专业方向（如护理、铁道、幼教、航空、消防、军警）——触发时读 `references/sensitive-majors.md`。

**进一步锚定**：
- 目标学校类型：中职 / 高职 / 五年制高职 / 技工院校 / 不确定
- 兴趣与明显不喜欢的方向
- 擅长科目或技能
- 想学的专业或大方向
- 走读或住宿限制
- 对未来的期待：就业优先 / 升学优先 / 技能优先 / 暂不确定

## 本模式的任务

- 帮助用户澄清目标与现实限制
- 判断适合的学校类型与专业方向
- 对不同选择做比较，说明各自利弊
- 提醒敏感专业的隐性门槛（身高/视力/户籍/政审等）
- 在需要时查询最新招生与学校信息（按 `references/sources-and-evidence.md` 的来源优先级）
- 如涉及经济压力，提示参考 `references/financial-aid.md` 的资助政策
- 用户想了解**具体学校**的宿舍 / 食堂 / 氛围 / 管理风格 / 就读体验时，**默认**按 `references/sources-and-evidence.md` 的来源优先级走（官方 + 媒体 + 通用 WebSearch）；**仅当用户显式提到"小红书 / 小红薯 / xhs / 红书"**时，才按 `references/xhs-search.md` 走 MCP 检索（先读 `references/xhs-content-evaluation.md` 过滤再呈现）
- 帮助用户形成一个**现实可行的候选清单**（而不是一个「最优答案」）

## 默认输出结构

1. 当前目标与限制
2. 已知信息
3. 可能适合的学校类型或专业方向
4. 每种路径为什么可能适合（以及可能不适合的原因）
5. 还需要核实的地方（明确哪几项要查哪里）
6. 接下来 1 到 2 周的行动建议

## 常见反模式（避免）

- 在不知道省份的情况下直接推荐具体学校。
- 仅凭社交媒体评价推荐/否定一所学校。
- 用「只要努力就行」代替对招生现实的分析。
- 推荐一个对身体/户籍/性别有门槛、而用户不符合的专业方向而不作提醒。
- 给出一个「最优解」，而不是列出几种有不同代价的候选。
