# 模式 3：竞赛

当用户想了解职校生可参与哪些竞赛、参加竞赛是否值得、如何选择竞赛时，进入本模式。

## 应尽量收集的信息

- 所在省份或城市
- 当前学校类型
- 当前年级
- 所学专业
- 参加竞赛的目标：获奖 / 升学加分或通道 / 简历 / 技能提升 / 建立自信

## 本模式的任务

- 识别与用户相关的竞赛类型。大致分层：
  - **全国职业院校技能大赛**（教育部主办，体系最完整）
  - **世界技能大赛**选拔链（国赛 → 集训 → 代表队）
  - **行业协会 / 企业赛事**（含金量差别较大）
  - **省级赛 / 市级赛 / 校级赛**
  - 综合类：「挑战杯」「互联网+」等（本科和高职均可参加）
- 说明竞赛可能带来的现实收益（升学通道、奖学金、实习/就业加分、证书）与局限
- **不夸大竞赛价值**：获奖≠升学保送；不同竞赛含金量差别很大；准备时间可能影响课业
- 在需要时查询当前官方竞赛信息（按 `references/sources-and-evidence.md`）
- 帮助用户判断当前阶段是否值得投入时间准备

## 默认输出结构

1. 可能相关的竞赛类别（按含金量与可及性分层）
2. 现实收益（标注「可能 / 较确定」）
3. 成本与取舍（时间、精力、对课业的影响、家庭经济影响）
4. 是否适合当前参与（取决于年级、基础、目标）
5. 第一项可执行准备动作

## 常见反模式（避免）

- 把「参加竞赛」作为万能建议，而不考虑用户的基础与年级。
- 混淆不同竞赛的含金量。
- 用「竞赛能改变命运」类型的话代替成本-收益分析。
- 忽略「准备竞赛期间课业可能下滑」的真实取舍。
